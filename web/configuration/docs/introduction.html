Introduction

'Oh no - not another PHP-Framework', you would say.

BUT - what if this framework is very much better to handle than any other framework?
BUT - what if this framework is a framework which you can develop to your own needs?
BUT - what if you can use other php-libraries like they are in this framework?
etc...

The main reason why i developed this framework is that there are so many unnecessary overhead and functions which i dont want to use but have to install with some framework so that my project raises to a size which is not rellay applicapable to the short project i want to build.

Second i was disappointed to see that every framework i wanted to use could be done only with one project.

THIS framework can be installed everywhere and can be used by more than one project.

And more:

Let's say you need a barchart-libary in your project.

There are many of them out there which are really useful.

But - including them in your project could cause a problem with pathes and/or other variables/libraries.

So - wouldn't it be nice to include this library into your normal framework?

With THIS it can be done in a very simple way.
